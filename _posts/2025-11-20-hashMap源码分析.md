## 一、简介

非线程安全

jdk1.7底层实现数组+链表，新节点插入采用头插法

![jdk1.8 之前的内部结构-HashMap](https://raw.githubusercontent.com/chen-1110/image/main/jdk1.7_hashmap.png)

jdk1.8底层实现数组+链表/红黑树，新节点插入采用尾插法（7上8下），当链表长度大于8且数组长度大于64时会将链表转为搜索效率更高的红黑树

![jdk1.8之后的内部结构-HashMap](https://raw.githubusercontent.com/chen-1110/image/main/jdk1.8_hashmap.png)

数组默认初始化大小为16，每次扩充容量*2



## 二、成员属性分析



```java
public class HashMap<K,V> extends AbstractMap<K,V> implements Map<K,V>, Cloneable, Serializable {
    // 序列号
    private static final long serialVersionUID = 362498820763181265L;
    // 默认的初始容量是16
    static final int DEFAULT_INITIAL_CAPACITY = 1 << 4;
    // 最大容量
    static final int MAXIMUM_CAPACITY = 1 << 30;
    // 默认的负载因子
    static final float DEFAULT_LOAD_FACTOR = 0.75f;
    // 当桶(bucket)上的结点数大于等于这个值时会转成红黑树
    static final int TREEIFY_THRESHOLD = 8;
    // 当桶(bucket)上的结点数小于等于这个值时树转链表
    static final int UNTREEIFY_THRESHOLD = 6;
    // 桶中结构转化为红黑树对应的table的最小容量
    static final int MIN_TREEIFY_CAPACITY = 64;
    // 存储元素的数组，总是2的幂次倍
    transient Node<k,v>[] table;
    // 一个包含了映射中所有键值对的集合视图
    transient Set<map.entry<k,v>> entrySet;
    // 存放元素的个数，注意这个不等于数组的长度。
    transient int size;
    // 每次扩容和更改map结构的计数器
    transient int modCount;
    // 阈值(容量*负载因子) 当实际大小超过阈值时，会进行扩容
    int threshold;
    // 负载因子
    final float loadFactor;
}
```

比较重要的有

1.loadFactor，负载因子默认0.75，当元素数量size大于数组容量*loadFactor会触发扩容

2.threshold 等于容量*loadFactor

3.Node<K,V>table数组，存储数组，Node是内部类，内部属性见下

```java
static class Node<K,V> implements Map.Entry<K,V> {
       final int hash;// 哈希值，存放元素到hashmap中时用来与其他元素hash值比较
       final K key;//键
       V value;//值
       // 指向下一个节点
       Node<K,V> next;
}
```

可以比较明显看到这是个链表类结构，jdk1.8后当容量增多发生树化后，树型数据存储在其子类TreeNode里，内部属性见下

```java
static final class TreeNode<K,V> extends LinkedHashMap.Entry<K,V> {
        TreeNode<K,V> parent;  // red-black tree links
        TreeNode<K,V> left;
        TreeNode<K,V> right;
        TreeNode<K,V> prev;    // needed to unlink next upon deletion
        boolean red;
}
```

这里有一个比较有趣的细节是，treeNode作为HashMap类里的内部类继承了LinkedHashMap的Entry内部类，而这个Entry内部类继承了HashMap类里的内部类Node类，这种交叉继承的反直觉逻辑其实比较好解释，因为treeNode需要树能力+节点顺序能力+链能力，Node类需要链能力，Entry需要节点顺序能力+链能力，所以出现了这种继承顺序Node->Entry->TreeNode

## 三、源码分析

### 懒汉式构造

hashMap默认创建时不会初始化（除了用另一个集合左右创建参数等特殊情况），只会记录下加载因子，在第一次put时才初始化空间，可以参考下面源码

```java
// 默认构造函数。
public HashMap() {
    this.loadFactor = DEFAULT_LOAD_FACTOR; // all   other fields defaulted
}
// 包含另一个“Map”的构造函数
public HashMap(Map<? extends K, ? extends V> m) {
    this.loadFactor = DEFAULT_LOAD_FACTOR;
    putMapEntries(m, false);//下面会分析到这个方法
}
// 指定“容量大小”的构造函数
public HashMap(int initialCapacity) {
    this(initialCapacity, DEFAULT_LOAD_FACTOR);
}

// 指定“容量大小”和“负载因子”的构造函数
public HashMap(int initialCapacity, float loadFactor) {
     if (initialCapacity < 0)
             throw new IllegalArgumentException("Illegal initial capacity: " + initialCapacity);
     if (initialCapacity > MAXIMUM_CAPACITY)
         initialCapacity = MAXIMUM_CAPACITY;
     if (loadFactor <= 0 || Float.isNaN(loadFactor))
         throw new IllegalArgumentException("Illegal load factor: " + loadFactor);
     this.loadFactor = loadFactor;
     // 初始容量暂时存放到 threshold ，在resize中再赋值给 newCap 进行table初始化
     this.threshold = tableSizeFor(initialCapacity);
}
```

### put方法

put()方法调用putVal()实现，putVal有5个入参，分别是

1.key的hash值（hashcode加一些位运算扰动使分布更分散）

2.key

3.value

4.onlyIfAbsent，bool值，用来判断当出现相同元素是否需要修改

5.evict，bool值，是否要类似lru的迭代更替老元素，这是为LinkedHashMap预留的字段，开发者可以实现指定函数为map增加类似lru的能力

具体逻辑注释：

```java
    /**
     * Implements Map.put and related methods.
     *
     * @param hash hash for key
     * @param key the key
     * @param value the value to put
     * @param onlyIfAbsent if true, don't change existing value
     * @param evict if false, the table is in creation mode.
     * @return previous value, or null if none
     */
    final V putVal(int hash, K key, V value, boolean onlyIfAbsent,
                   boolean evict) {
        Node<K,V>[] tab; Node<K,V> p; int n, i;
        if ((tab = table) == null || (n = tab.length) == 0)
            // 首次put，初始化
            n = (tab = resize()).length;
        if ((p = tab[i = (n - 1) & hash]) == null)
            // index上无元素，直接插入
            tab[i] = newNode(hash, key, value, null);
        else {
            Node<K,V> e; K k;
            if (p.hash == hash &&
                ((k = p.key) == key || (key != null && key.equals(k))))
                // 头结点key相同，记录下来等后面更新
                e = p;
            else if (p instanceof TreeNode)
                // index处是一颗树，走树更新逻辑
                e = ((TreeNode<K,V>)p).putTreeVal(this, tab, hash, key, value);
            else {
                // index是链表，遍历链表
                for (int binCount = 0; ; ++binCount) {
                    if ((e = p.next) == null) {
                        //没有相同key的元素，新建节点
                        p.next = newNode(hash, key, value, null);
                        if (binCount >= TREEIFY_THRESHOLD - 1) // -1 for 1st
                            // 达到树化阈值，走树化逻辑
                            treeifyBin(tab, hash);
                        break;
                    }
                    if (e.hash == hash &&
                        ((k = e.key) == key || (key != null && key.equals(k))))
                        break;
                    p = e;
                }
            }
            if (e != null) { // existing mapping for key
                // 有相同元素，根据入参判断是否更新
                V oldValue = e.value;
                if (!onlyIfAbsent || oldValue == null)
                    e.value = value;
                //预留后处理方法，无实现逻辑
                afterNodeAccess(e);
                return oldValue;
            }
        }
        ++modCount;
        if (++size > threshold)
            //增加新节点后到达扩容阈值，扩容
            resize();
        //预留后处理方法，linkedHashMap可实现迭代更新逻辑
        afterNodeInsertion(evict);
        return null;
    }
```



putTreeVal写入树节点逻辑注释如下：

```java
/**
         * Tree version of putVal.
         */
        final TreeNode<K,V> putTreeVal(HashMap<K,V> map, Node<K,V>[] tab,
                                       int h, K k, V v) {
            Class<?> kc = null;
            boolean searched = false;
            TreeNode<K,V> root = (parent != null) ? root() : this;
            for (TreeNode<K,V> p = root;;) {
                int dir, ph; K pk;
                if ((ph = p.hash) > h)
                    // 左移
                    dir = -1;
                else if (ph < h)
                    // 右移
                    dir = 1;
                else if ((pk = p.key) == k || (k != null && k.equals(pk)))
                    // 出现key相同节点，返回相同节点
                    return p;
                else if ((kc == null &&
                          (kc = comparableClassFor(k)) == null) ||
                         (dir = compareComparables(kc, k, pk)) == 0) {
                    //hash相同，不可比较，需要遍历左右子树
                    if (!searched) {
                        TreeNode<K,V> q, ch;
                        searched = true
                        //遍历左右子树，找是否有相同
                        if (((ch = p.left) != null &&
                             (q = ch.find(h, k, kc)) != null) ||
                            ((ch = p.right) != null &&
                             (q = ch.find(h, k, kc)) != null))
                            return q;
                    }
                    // 虽然hash相同，但是上面遍历左右子树没找到相同的，任意往左或往右皆可，tieBreakOrder方法返回一个一定会存在的左右值，
                    dir = tieBreakOrder(k, pk);
                }

                //上面逻辑走到底没找到相同的，走插入逻辑
                TreeNode<K,V> xp = p;
                if ((p = (dir <= 0) ? p.left : p.right) == null) {
                    Node<K,V> xpn = xp.next;
                    TreeNode<K,V> x = map.newTreeNode(h, k, v, xpn);
                    if (dir <= 0)
                        xp.left = x;
                    else
                        xp.right = x;
                    xp.next = x;
                    x.parent = x.prev = xp;
                    if (xpn != null)
                        ((TreeNode<K,V>)xpn).prev = x;
                    moveRootToFront(tab, balanceInsertion(root, x));
                    return null;
                }
            }
        }
```

### get方法

```java
public V get(Object key) {
    Node<K,V> e;
    return (e = getNode(hash(key), key)) == null ? null : e.value;
}

final Node<K,V> getNode(int hash, Object key) {
    Node<K,V>[] tab; Node<K,V> first, e; int n; K k;
    if ((tab = table) != null && (n = tab.length) > 0 &&
        (first = tab[(n - 1) & hash]) != null) {
        // 数组元素相等
        if (first.hash == hash && // always check first node
            ((k = first.key) == key || (key != null && key.equals(k))))
            return first;
        // 桶中不止一个节点
        if ((e = first.next) != null) {
            // 在树中get
            if (first instanceof TreeNode)
                return ((TreeNode<K,V>)first).getTreeNode(hash, key);
            // 在链表中get
            do {
                if (e.hash == hash &&
                    ((k = e.key) == key || (key != null && key.equals(k))))
                    return e;
            } while ((e = e.next) != null);
        }
    }
    return null;
}
```
